# Dinner Deck

A family meal planning app for a family of four. Pick five recipes for the work week (Monday–Friday), scale servings per meal, and generate a grocery list grouped the way you'd actually walk a store — produce, meat, dairy, and so on.

Built to run on lightweight hardware (a Raspberry Pi 3 alongside other small Flask apps), so the stack has no build step and no JS framework.

## Features

- **Recipe management** — add, edit, delete recipes with structured ingredients (item, amount, unit), cuisine type, cook time, and tags
- **Meal generator** — randomly picks 5 recipes (Mon–Fri) from your recipe pool
- **Exclusion filters** — exclude recipes by tag when generating a plan (e.g. `shrimp`, `shellfish`, `peanuts`)
- **Per-meal serving scaling** — scale any single meal up or down (e.g. drop to 2 servings for a night when a kid's away) without touching the recipe's base serving size
- **Grocery list generation** — merges ingredient quantities across the week's meals and groups them by grocery store section (Produce, Meat & Seafood, Dairy & Eggs, etc.)
- **PWA / offline support** — installable on a phone home screen; the current week's plan and grocery list are cached so they're usable at the store without a connection back to the Pi
- **100 pre-loaded recipes** — healthy, budget-friendly, under 30 minutes, spanning Asian, Tex-Mex, Mediterranean, and American cuisines, no dietary restrictions

## Stack

- **Backend:** Python 3 / Flask / Flask-SQLAlchemy
- **Database:** SQLite (`instance/dinner_deck.db`, gitignored)
- **Frontend:** Vanilla HTML/CSS/JS, no build step
- **Offline:** Service worker (`static/js/sw.js`) — network-first for the app shell, cache-fallback for both the shell and `/api/*` GET responses when offline

## Data model

- **Recipe** — name, cuisine type, cook time, base servings, instructions, tags
- **Ingredient** — normalized ingredient name + grocery category (enables quantity merging and store-section grouping)
- **RecipeIngredient** — join table: amount + unit of an ingredient within a recipe
- **Tag** — descriptive labels used for exclusion filtering (not dietary-restriction enforcement)
- **WeeklyPlan** — a Monday–Friday plan
- **WeeklyPlanMeal** — one recipe assigned to one day, with an optional `servings_override` for scaling
- **GroceryList / GroceryListItem** — generated by merging `RecipeIngredient` amounts (scaled per meal) across a plan's meals

## Setup

```bash
git clone https://github.com/chrislockejr/dinner-deck.git
cd dinner-deck
python3 -m venv venv
source venv/bin/activate
pip install -r requirements.txt
python3 migrate_seed.py        # loads the 100 seed recipes
python3 migrate_categories.py  # categorizes ingredients for grocery grouping
python3 app.py
```

The app runs on **port 5050** (not 5000 — on macOS, AirPlay Receiver holds port 5000; kill it with `lsof -ti:5000 | xargs kill -9` if you ever want 5000 back). Visit `http://localhost:5050`.

## Deploying to a Raspberry Pi

This is intended to run alongside other lightweight Flask apps on a Pi 3 (1GB RAM):

1. Clone the repo onto the Pi.
2. Create a venv and install `requirements.txt`.
3. Run the two migration scripts once to seed the database.
4. Use a production WSGI server instead of the Flask dev server, e.g.:
   ```bash
   pip install gunicorn
   gunicorn -w 2 -b 0.0.0.0:5050 app:app
   ```
5. Use a process manager (systemd service or `start.sh` + `nohup`) to keep it running across reboots.

The PWA service worker caches the app shell and the latest plan/grocery-list API responses, so the phone can still show the current week's grocery list even if it can't reach the Pi (e.g. you're on cellular at the store, off your home WiFi).

## Adding a custom recipe ingredient

New ingredients created through the UI default to the "Other" grocery category. To assign a proper category, add an entry to `CATEGORY_MAP` in `migrate_categories.py` and re-run it — it's safe to run repeatedly.
